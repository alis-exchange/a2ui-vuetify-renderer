# A2UI Vue Renderer

The A2UI Vue Renderer is a high-performance, robust rendering layer that bridges the [Agent-to-User Interface (A2UI) Protocol v0.9](https://a2ui.org/) with the Vue 3 ecosystem. It dynamically translates A2UI JSON messages into interactive, native Vue interfaces utilizing the comprehensive **Vuetify 4** component library.

This library is designed for App Developers and Agent Developers who need a seamless Vue integration to render rich, interactive UIs generated by intelligent agents.

## Core Dependencies
* **Vue 3**: Utilizes the Composition API and reactivity system (`ref`, `provide/inject`) to bridge agent state to DOM updates.
* **Vuetify 4**: The core library used to implement the A2UI component catalog natively, utilizing its rich components and theming engine.
* **@a2ui/web_core**: The framework-agnostic A2UI message processor, state manager, data binding, and expression evaluation library.

## Installation & Setup

1. **Install the renderer and its dependencies:**

```bash
npm install a2ui-vue-renderer @a2ui/web_core vuetify vue
# or
pnpm add a2ui-vue-renderer @a2ui/web_core vuetify vue
# or
yarn add a2ui-vue-renderer @a2ui/web_core vuetify vue
```

2. **Configure Vuetify and Vite:**

You must configure Vuetify in your host application. This renderer relies on Vuetify being installed and accessible via your app's standard plugin chain.

**`vite.config.ts`**
```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import vuetify from 'vite-plugin-vuetify'

export default defineConfig({
  plugins: [
    vue(),
    vuetify({ autoImport: true })
  ]
})
```

**`main.ts`**
```typescript
import { createApp } from 'vue'
import App from './App.vue'
import { createVuetify } from 'vuetify'
import 'vuetify/styles'
import '@mdi/font/css/materialdesignicons.css' // Required for icons

const vuetify = createVuetify()
const app = createApp(App)

app.use(vuetify)
app.mount('#app')
```

## Getting Started (Basic Setup)

To render A2UI surfaces, you need to instantiate the `MessageProcessor`, define your component catalog, and wrap your rendering area with the `A2UIProvider`.

Here is a complete example of setting up a mock A2UI surface in a Vue component (e.g., `App.vue`):

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue';
import { MessageProcessor, Catalog } from '@a2ui/web_core/v0_9';
import type { Action } from '@a2ui/web_core/v0_9/schema';
// Note: Depending on your import aliases, adjust the path to a2ui-vue-renderer exports
import { A2UIProvider, ComponentNode } from 'a2ui-vue-renderer';

// 1. Define how your app handles actions sent by the agent
const handleAction = (action: Action) => {
  console.log('Action triggered by A2UI component:', action);
  // Example: send this back to your agent backend via WebSocket
};

// 2. Initialize the Catalog and Processor
// We use the basic catalog URI as defined in the v0.9 spec
const mockCatalog = new Catalog('https://a2ui.org/specification/v0_9/basic_catalog.json', []);
const processor = new MessageProcessor([mockCatalog], handleAction);
const surfaceId = 'my-surface-1';
const isReady = ref(false);

onMounted(() => {
  // 3. Simulate receiving JSONL messages from an agent
  
  // Message 1: Create the surface
  processor.processMessages([{
    createSurface: {
      surfaceId,
      catalogId: 'https://a2ui.org/specification/v0_9/basic_catalog.json',
      theme: { primaryColor: '#6200EE' }
    }
  }]);

  // Message 2: Populate it with components
  processor.processMessages([{
    updateComponents: {
      surfaceId,
      components: [
        {
          id: 'root',
          component: 'Column',
          children: ['title', 'btn']
        },
        {
          id: 'title',
          component: 'Text',
          text: 'Hello from A2UI!',
          variant: 'h4'
        },
        {
          id: 'btn',
          component: 'Button',
          variant: 'primary',
          child: 'btn-text',
          action: { event: { name: 'greet' } }
        },
        {
          id: 'btn-text',
          component: 'Text',
          text: 'Click Me'
        }
      ]
    }
  }]);
  
  isReady.value = true;
});
</script>

<template>
  <v-app>
    <v-main class="pa-4">
      <div v-if="!isReady">Loading A2UI Surface...</div>
      
      <!-- 4. Wrap your rendering area in the A2UIProvider and mount the root node -->
      <A2UIProvider 
        v-else 
        :processor="processor" 
        :surface-id="surfaceId" 
        :on-action="handleAction"
      >
        <ComponentNode id="root" />
      </A2UIProvider>

    </v-main>
  </v-app>
</template>
```

## Advanced Usage

### The Component Registry

The A2UI renderer relies on a `ComponentRegistry` to map the string types provided by the agent (e.g., `"Button"`, `"TextField"`) to their corresponding native Vue/Vuetify components.

By default, installing the renderer plugin initializes the `defaultRegistry` which maps all basic A2UI v0.9 components to our internal Vuetify 4 wrappers.

If you need to inject **custom components** or override default ones, you can utilize the registry instance.

```typescript
import { defaultRegistry } from 'a2ui-vue-renderer';
import MyCustomVueMap from './components/MyCustomVueMap.vue';

// Register a new custom component that your agent can now send as `component: "GoogleMap"`
defaultRegistry.register('GoogleMap', MyCustomVueMap);

// Or override an existing default
import MyCustomButton from './components/MyCustomButton.vue';
defaultRegistry.register('Button', MyCustomButton);
```

### Accessing A2UI Context inside Custom Components

When building custom components, you can use the `useA2UI` composable to hook into the A2UI context, resolve bindings, update the data model, and send actions.

```vue
<!-- MyCustomVueMap.vue -->
<script setup lang="ts">
import { computed } from 'vue';
import { useA2UI } from 'a2ui-vue-renderer';
import type { ComponentModel } from '@a2ui/web_core/v0_9/state/component-model';

const props = defineProps<{
  node: ComponentModel;
}>();

const { resolveValue, sendAction, setData } = useA2UI();

// Resolve dynamic properties set by the agent
const lat = computed(() => resolveValue(props.node.properties.lat));
const lng = computed(() => resolveValue(props.node.properties.lng));

const handleMapClick = (eventData: any) => {
  // Update the shared A2UI data model immediately
  setData('/map/lastClick', eventData);

  // Send an action event back to the agent backend
  sendAction('map_clicked', { lat: eventData.lat, lng: eventData.lng });
}
</script>

<template>
  <div class="my-map-wrapper">
    <!-- Map implementation using `lat` and `lng` -->
  </div>
</template>
```

## Local Development & Contribution

If you want to contribute to the renderer or test it locally:

1.  **Clone the repository:**
    ```bash
    git clone <repository_url>
    cd vue-a2ui-renderer/renderer
    ```

2.  **Install dependencies:**
    ```bash
    pnpm install
    ```

3.  **Start the development server:**
    The project uses Vite. Starting the dev server will run the `App.vue` sandbox for manual testing.
    ```bash
    pnpm run dev
    ```

4.  **Run automated tests:**
    Vitest is used for component and unit testing.
    ```bash
    pnpm test
    # Or to see coverage:
    pnpm test -- --coverage
    ```
